# 🎬 Reels Trend Agent

Агент, который **каждое утро** собирает свежие вирусные Instagram Reels на
английском языке по темам **книги · успех · предпринимательство** и кладёт
отчёт в папку [`reports/`](reports/). Главная метрика — **как быстро ролик
набрал первый миллион просмотров**: отчёт отсортирован по ней.

## Как это работает

```
GitHub Actions (каждое утро, отчёт к ~07:30 МСК)
   └─► agent/collect_reels.py
         1. Поиск Reels по ключевым запросам трёх тем (Apify, поиск
            Instagram) — по 3 запроса на тему
         2. Перепроверка вчерашних кандидатов по прямым ссылкам, пока
            им не исполнится 7 дней (строит реальную кривую просмотров)
         3. Фильтры: только видео · не старше 7 дней · английский язык ·
            автор с аудиторией НЕ БОЛЬШЕ 100 тыс подписчиков
            (недостающие подписчики добираются скрейпером профилей)
         4. Замер скорости до 1 млн (см. «Методика» ниже)
         5. Отчёт → reports/ГГГГ-ММ-ДД.md (+ reports/latest.md)
         6. История замеров → data/reels_history.json
         7. Опционально — сводка в Telegram
```

**Фильтр по подписчикам — ключевой.** Миллион просмотров у канала с
многомиллионной аудиторией — это его норма, а не вирус. Агент ищет
ролики, которые разогнал алгоритм: у автора меньше 100 тыс подписчиков,
а просмотры кратно больше аудитории (колонка «×аудитория» в отчёте).
Поисковые запросы — словарь `SEARCH_QUERIES` в начале
[`agent/collect_reels.py`](agent/collect_reels.py), редактируйте под себя.
Дополнительные источники: `REELS_USE_WATCHLIST=1` (Reels с крупных
аккаунтов из списка `ACCOUNTS`, без фильтра по подписчикам) и
`REELS_USE_HASHTAGS=1` (ленты хэштегов).

## Быстрый старт

1. **Токен Apify.** Зарегистрируйтесь на [apify.com](https://apify.com)
   (есть бесплатный тариф) и скопируйте API-токен из
   *Settings → API & Integrations*.
2. **Секрет репозитория.** В GitHub: *Settings → Secrets and variables →
   Actions → New repository secret*, имя — `APIFY_TOKEN`.
3. **Влейте эту ветку в основную.** GitHub запускает cron-workflow только
   из ветки по умолчанию.
4. **Проверьте вручную.** *Actions → Daily Reels Report → Run workflow* —
   первый отчёт появится в `reports/` через несколько минут.

### Отчёт в Telegram (по желанию)

Чтобы сводка приходила в Telegram:

1. Создайте бота у [@BotFather](https://t.me/BotFather) и добавьте его токен
   секретом `TELEGRAM_BOT_TOKEN`.
2. Напишите боту любое сообщение (например, `/start`) — при первом запуске
   агент сам определит ваш `chat_id` и сохранит его в
   `data/telegram_chat_id.txt`. Задавать секрет `TELEGRAM_CHAT_ID` вручную
   не обязательно (но можно — он имеет приоритет).

## Методика: скорость до первого миллиона

Instagram не публикует историю просмотров, поэтому агент строит её сам:

- при каждом запуске он записывает текущие просмотры каждого ролика в
  `data/reels_history.json`;
- **✅ замерено** — ролик попал в историю ещё до миллиона, и момент
  пересечения вычислен интерполяцией между двумя реальными замерами;
- **≈ оценка** — ролик уже был за миллионом при первом обнаружении; время
  рассчитано по его средней скорости с момента публикации.

В первые дни работы почти все строки будут оценками — точность растёт по
мере накопления истории.

## Настройка

Переменные окружения (задаются в workflow или в *Settings → Variables*):

| Переменная | По умолчанию | Что делает |
|---|---|---|
| `REELS_MAX_FOLLOWERS` | `100000` | максимум подписчиков у автора ролика |
| `REELS_MAX_AGE_DAYS` | `7` | максимальный возраст ролика |
| `REELS_SEARCH_PAGES` | `2` | глубина поиска (страниц выдачи на запрос) |
| `REELS_USE_WATCHLIST` | выкл. | `1` — добавить Reels крупных аккаунтов из `ACCOUNTS` |
| `REELS_PER_ACCOUNT` | `5` | Reels с аккаунта watchlist (если включён) |
| `REELS_USE_HASHTAGS` | выкл. | `1` — добавить ленты хэштегов |
| `REELS_RESULTS_LIMIT` | `10` | постов на хэштег (если хэштеги включены) |
| `REELS_RISING_MIN_VIEWS` | `100000` | порог просмотров для раздела «на подлёте» |
| `REELS_RISING_MIN_VPH` | `10000` | порог темпа (просмотров/час) для «на подлёте» |
| `REELS_TRACK_MIN_VIEWS` | `50000` | с каких просмотров кандидат попадает в ежедневный трекинг |

Время запуска — строка `cron` в
[`.github/workflows/daily-reels-report.yml`](.github/workflows/daily-reels-report.yml)
(указывается в UTC; стоит `45 3 * * *` = 06:45 МСК с запасом, потому что
GitHub задерживает cron-запуски до 40+ минут). Workflow также запускается
при каждом пуше в `main`, затрагивающем файлы агента, — так удобно
проверять изменения.

**Расход Apify.** Скрейперы тарифицируются за собранные элементы:
9 запросов × ~2 страницы ≈ 200–250 элементов в день, плюс дозапросы
профилей и трекинг кандидатов (обычно ещё 10–50). Если лимита не
хватает, уменьшите `REELS_SEARCH_PAGES` или число запросов в
`SEARCH_QUERIES`.

**Диагностика.** Каждый запуск пишет `data/debug_last_run.json`: сколько
элементов вернул Apify, что отсеяли фильтры, статус отправки в Telegram.

## Ограничения

- Instagram отдаёт публично только текущее число просмотров, поэтому для
  роликов, «пойманных» уже после миллиона, время до 1 млн — оценка.
- Язык определяется по подписи ролика; ролики без текста в подписи
  считаются английскими (аудио не анализируется).
- Скрейпинг зависит от Apify: если Instagram меняет разметку, актёр
  обновляется командой Apify, но возможны разовые сбои — workflow в этот
  день просто завершится с ошибкой и повторит попытку назавтра.
